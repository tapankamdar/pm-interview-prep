# PM Framework Guide

The most common PM interview mistake at senior levels: deploying a framework when the interviewer wanted your thinking, not your template. Guide the user toward frameworks that help and away from frameworks that signal they learned interviewing from a prep course.

## The Core Test

Before recommending any framework, ask: **"Does this framework make the answer more specific to this company and role — or less?"**

If the framework adds structure without adding company-specific insight, it's a liability above IC level. A Director or VP using CIRCLES on a product sense question signals they're pattern-matching to interview prep, not thinking about the actual problem.

## Quick Routing

| Situation | Guidance |
|---|---|
| Product design question, IC/APM level | CIRCLES is acceptable as a scaffold — but still needs user specificity |
| Product design question, Sr. PM / Director+ | Skip CIRCLES. Lead with user insight, build your own structure |
| Metrics question | No standard framework — build a diagnostic tree specific to the product |
| Prioritization question | RICE/ICE are fine for framing, but the insight has to be yours — why this score for this company |
| North star metric | Don't use a framework — show you've thought about what actually drives this business |
| Strategy / vision | 3-pillar structure works, but each pillar must be grounded in a company-specific fact |
| Estimation | Decomposition is the method — no named framework needed, just label your assumptions |
| Behavioral | STAR is the structure — but "earned secret" is what makes it differentiated |

## The Framework Liability Signal

Flag these patterns as warning signs in any answer:

- Opening with the framework name ("I'll use CIRCLES here...")
- Treating all framework steps as equal even when some don't apply
- Arriving at the same answer structure regardless of the specific company or product
- Sounding like a prep guide rather than a practitioner

When these appear, surface them: "This answer is following the framework faithfully — but it could have been written for any company. What's the insight that's specific to [this company's] situation?"
